Hexo 簡介

Hexo 介紹

Hexo 是一個建立靜態網誌的框架(基於Node.js)，主要使用Markdown語法格式。這篇文章主要紀錄實際使用方法與建置步驟。配合Github Page可以很方便的佈署網誌到Github上。

  • 需求(需事先安裝or註冊): nodejs, Git, Github

建置步驟

初始化專案: 置換成資料夾名稱

創建完成後，可以看到以下資料夾格式

  • _config.yml: 各種網誌的設定配置
  • /scaffolds: 文章的模板，創建文章時會使用對應名稱的模板，進行格式初始化
  • /themes: 整個網誌的 theme，可以到官網下載
  • /source: 用來存放，草稿、文章的位置
    • _drafts:存放草稿
    • _posts:存放文章

將草稿發布至文章
也可以直接使用此指令創建文章

佈署與發佈

產生 html 檔案

因為撰寫的主要為 .md 格式，需要以下指令將 .md -> .html 與 .css 等網頁檔案格式

佈署到 Github 上

(第一次)
需調整 _config.yml 檔案中的 Deploy 屬性內的值，注意 branch 名稱需要先到 github 上查看主要的 branch 名稱。(Github預設已經從 master -> main)

產生對應的格式檔案後，接著就可以將網誌 push 到 github 上。

整體步驟為

  • 創建草稿 -> 發佈到 publish 區 -> 產生 html等檔案 -> 佈署到 github
