DataStructure 1 - Intro

Data Structure 系列

This is Series of Data Structure.

因為最近非本科轉換領域的關係，以此來記錄學習的歷程，主要學習的資料來源為 Udemy 上的課程。

  • 名稱: Mastering Data Structures & Algorithm using C/C++
  • 老師: Adbul Bari
  • 時長: 58 hr 30 min

0. Introduction 基本介紹

資料結構表達出如何有組織地將資料放置於記憶體中。主要分類為:

  • 實際資料結構(Physical Data Structure)
    • 陣列 Array
    • 串列 Linked List
  • 邏輯資料結構(Logical Data Structure)
    • 堆疊 Stack
    • 駐列 Queue
    • 樹 Trees
    • 圖 Graph
    • 哈希表 Hashing
    • 遞迴 Recursion

好的資料結構規劃，搭配上演算法可以讓資料存取上更加的有效率。最後會簡單說明上方這些資料結構的基本特性。

學習資料結構的等級

  1. 知道資料結構如何運作
  2. 知道執行時的消耗、可以有效的分析該資料結構的效率
  3. 可以自行創建有利於情況的資料結構

這系列學習筆記使用 C/C++ 來實作資料結構

1. 介紹資料儲存

在世界上的資料，目前有四種主要的處理方式，其中對應到不同的技術與演算法。分別為:

資料結構 (Data structure)

  • 如何有效率的儲存/組織主記憶體上的資料
  • 如何有效率的存取主記憶體中的資料

資料庫 (Storage)

  • 如何將資料放置於硬碟 (Hard Disk) 中，並採用標準的格式，使主記憶體更容易從第三類儲存設備中存取資料
  • 重點在於如何設計資料庫結構

資料倉儲 (Data Warehouse)

  • 儲存大量不常使用的資料 (Legacy data or historical data)
  • 由大量的硬碟陣列 (Array of Disk)磁片儲存
  • 目的儲存10年、50年以上
  • 專門處理此種資料的演算法: 資料探勘 (Data Mining Algorithm)

大數據(Big Data)

  • 藉由網路為主要的媒介，蒐集各式各樣的資料並處理
  • 主要演算法: 機器學習

一般程式執行時，處理的流程為: 硬碟(Hard Disk) > 主記憶體 > CPU 暫存器 > CPU 處理。

2. 主記憶體 Stack & Heap

記憶體最小單位 Bytes，每一個 Bytes 都有自己的位址編號(線性唯一編號)。現今的記憶體會進行主動切割為 Segement 當作一個可管理的記憶體單元(64kB = 64 * 1024 = 65536)

程式碼如何使用記憶體

一個 Segement (0 ~ 65535)，主要被分為三個部分。其中程式碼中變數所佔的 Bytes 數是根據編譯器與作業系統來決定。

  • Heap
  • Stack
  • Code Section

Stack

  1. 使用範圍: 記憶體編號低到高
  2. 函數在**編譯時期(Compiler time)**決定需要多少 Bytes 的記憶體空間
  3. 當函數呼叫時，自動分配；當函數結束時，自動釋放
  4. 程式可直接存取 (stack 與 code section 資源)

Heap

名稱常常同時用於兩個情境(很容易混淆)

  • 被組織如同塔狀的結構
  • (此處用法)沒有組織，但看起來像塔狀的結構
  1. 使用範圍: 記憶體編號高到低
  2. 無組織
  3. Heap Memory 被稱為 Resources(資源)
    需要時可動態拿取；不需要時可以自行釋放
  4. 程式無法直接存取 Heap 記憶體資源
    • 需要透過 Pointer 指標存取
  5. 需要使用 C++(new)、C(malloc) 來創建空間
  6. 不需要使用時，需要透過 C++(delete [])、C(free()) 來釋放空間
    • 可能造成位址丟失，該記憶體資源將永久卡住無法使用(Memory Leak)
    • 如果無釋放，當資源塞滿 Heap 會導致程式發生不可預期的情況

Code Section

  • 根據讀取的程式大小來動態調整
  • 放置函數的程式碼(e.g main function)

操作記憶體模式有

  • 靜態分配記憶體(Static Allocation): 只有 Stack
  • 動態分配記憶體(Dynamic Allocation): 可使用 Stack、Heap

3. 實體與邏輯資料結構

實體資料結構(Physical data structure) ，決定實際資料儲存在哪一塊記憶體，主要儲存方式為

Array

  • 固定大小(Size)
  • 可以被創建在 Stack 與 Heap (使用 pointer)

Linked List

  • 動態資料結構
  • 可動態調整大小
  • 只能被創建在 Heap (使用 pointer)
    • 在 Stack 中創建 Head 的指標指向第一個節點

邏輯資料結構(Logical data structure) ，決定依照什麼原則存取資料。(底層依然是由 array, Linked List 來實作)。主要方式為

線性資料結構(Linear)

  • Stack: 原則 LIFO
  • Queue: 原則 FIFO

非線性資料結構(non-Linear)

  • Tree: 原則 階層式
  • Graph: 原則 節點、連結式

表格式資料結構(Tabular)

  • Hash Table: 原則 Hash function

4. ADT 抽象資料結構

將較為底層的操作(如:位元等級的運算)隱藏，讓使用者只需要使用規定好的介面對資料進行存取異動即可。一般根據資料儲存資訊，資料操作來理解。

資料型別(Data type)

顯示資料(Representation of Data)

  • 資料如何被儲存
  • 舉例: int x；(被儲存 4 bytes = 32 bits)

資料操作(Operation on Data)

  • 可以對該資料進行何種操作
  • 舉例: -, +, *, /, %, ++, --

舉例: ADT(抽象型別)

顯示資料

  • 儲存資料的空間 int *A (儲存在 Heap 動態分配)
  • 最大容量(Capacity) int maxSize
  • 目前存放多少資料(size) int length

資料操作

  • 新增
    • add(element) / append(element)
    • add(index, element) / insert(index, element)
  • 刪除
    • remove(index)
  • 設置
    • set(index, element) / replace(index, element)
  • 搜尋
    • get(index)
    • search(key) / contains(key)
  • 排序
    • sort()
