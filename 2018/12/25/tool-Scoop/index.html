Scoop 套件管理軟體

Scoop Windows套件管理軟體

一個 windows 的軟體管理工具，用於安裝各式各樣的環境，類似於 Linux 的 apt-get。
相似的軟體有 :

  • Chocolatey
  • Ninite
  • CoApp。

Scoop 專注於在 Command-line 的使用，可以安裝軟體不需要一直挑選網頁下載。

快速安裝

要求 Requirement

  • PowerShell 3
  • 允許 PowerShell 可執行本地腳本

Installing Scoop

確認上方無誤後，在 powershell 中打入安裝指令。

如果想要安裝到指定的目錄 (如 : D: \Applications\Scoop)

Scoop 的使用

主要常用的指令有下列幾個 :

  • scoop help
    • 查看有哪些指令可用
  • scoop install {app name}
    • 安裝 app
  • scoop search {app name}
    • 搜尋確認是否有此 app ，或觀察目前版本
  • scoop update
    • 更新 scoop ，後方可加上 app 名稱，表示更新 app。 如果使用 * 則是安裝的全部都更新
  • scoop cleanup {app name}
    • 刪掉目前沒用或較舊的 app 版本
  • scoop uninstall {app name}
    • 刪掉現有的 app
  • scoop uninstall scoop
    • 刪掉他自己

Concepts

Apps

Scoop 使用名稱 “App” 表示一個應用程式或是需要被安裝的程式。一個 “App”應該是一個獨立的單元，也就是說一個”App”可能包含一個或是多個可執行的腳本。

Buckets

一個 Bucket 像是一個容器，裡面有包含很多個可安裝的 “App”。準確來說是一個 github 的 repository，其中包含用來描述安裝 App 的 JSON 語法，

Scoop 有一個主要的 Bucket，當你使用scoop install <app> 會先搜尋主要的 bucket，當然還有一個外部的 Bucket，更甚至你可以自己做自己的 bucket。
語法

  • 調查已知的 bucket，可以直接觀察檔案 “buckets.json”，或是打入指令

  • 安裝其他的 bucket 到電腦中

  • 創建自己的 bucket
    使用 github 的 repository，
  1. 創建一個新的 github repository
  2. 加入 app 到你的 bucket 中
  3. 設定你的 scoop 去連結你自己新建立的 bucket :
  4. 檢查是否成功`
