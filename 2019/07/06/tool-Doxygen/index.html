Doxygen 文檔建立系統

Doxygen介紹

Doxygen 是一種開源跨平台的文檔建立系統，(類似 javaDoc)。完全支持 C/C++、Java、Object-C

優點

  • 可從一組帶有註解的程式中生成 HTML 格式或是 LATEX 格式的文檔，亦可產生 pdf Unix man 等手冊文件。因為是從程式碼註解中直接提取文件，容易保持格式的一致性。
  • 可提取程式碼的結構，通過包含依賴圖(Dependency Graphs)、繼承圖(inheritance Diagram)、協作圖(Collaboration Diagram)等自動生成的圖，可使不同程式模組關係視覺化。
  • 可輸入格式: html, xml, LaTeX, PDF, RTF, Unix man

官網: Doxygen

基礎操作

安裝 Doxygen (Ubuntu)

sudo apt-get install doxygen

指令

Doxygen 可分為兩部分

  1. 特定格式註解的寫法(主要)
  2. 利用Doxygen工具來產生文件

產生一份 Doxygen 預設的 Config 檔案，並以此檔案來進行修改

  • doxygen -g [config-file]: 會產生Doxyfile(一個檔名叫做 “[config-file]”的預設 config 檔)
  • doxygen -u [config-file]: 更新舊的配置文件
  • doxygen [config-file]: 根據Doxyfile樣式，生成文檔

生成不同風格的文件，如 rtf, html, LaTeX

  • RTF格式: doxygen -w rtf styleSheetFile
  • HTML格式: doxygen -w html headerFile footerFile styleSheetFile [config-file]
  • LaTeX格式: doxygen -w latex headerFile StyleSheetFile [config-file]

註解格式

一般來說，要為每個 class 和 class 中重要的成員函數增加短註解和長註解。

  • 短註解給出類別或函數的基本訊息的描述
  • 長註解則是更加完整的描述。
  1. 類別的短註解和長註解與成員函數的簡短描述，放在標頭檔中。
  2. 成員函數的長註解將出現在成員函數的實現出現的位置
